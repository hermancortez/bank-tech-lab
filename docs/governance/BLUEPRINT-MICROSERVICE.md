# Blueprint de microservicio

Cada servicio debe declarar responsabilidad, dueño, API/eventos, SLO, dependencias y datos. La base incluye Java 21, estructura hexagonal, README, Dockerfile, manifiestos Kubernetes, Actuator y documentación del servicio.

Antes de separar un microservicio se valida autonomía del dominio, despliegue independiente y costo operacional.
